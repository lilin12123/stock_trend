# 项目功能使用说明书

## 1. 这是什么

这是一个用于盘中盯盘的本地控制台。

它会持续监控你关注的股票，在满足预设条件时给出提示，帮助你更快发现：

- 开盘后是否出现突破
- 价格是否明显偏离均价
- 波动是否突然放大
- 是否出现超买超卖、回踩确认、量价背离等信号

它的定位是 **提示工具**，不是自动交易系统，不会帮你下单。

当前版本还有两个重要特点：

- 同一根 K 线最终最多只会显示 `1` 条信号
- 如果同一根 K 线上同时出现 `up` 和 `down` 触发，它们会先互相抵消，再决定是否显示以及显示什么等级

## 2. 适合谁用

- 需要盯盘但不想一直手动刷行情的人
- 想把一组股票放进监控池，等系统自动提醒的人
- 想比较不同策略参数效果的人
- 想给不同账号设置不同观察股票和提醒方式的小团队

## 3. 核心功能一览

- 实时监控股票
- 自动识别盘中异动信号
- 在网页里集中查看信号和图表
- 给不同用户设置不同股票池
- 给不同用户设置不同提醒方式
- 支持历史日期回测
- 支持保存常用策略配置

## 4. 登录与权限

系统支持两类账号：

- 管理员
- 普通用户

管理员比普通用户多出来的能力主要是：

- 管理全局股票池
- 创建用户
- 重置其他用户密码

普通用户主要使用自己的功能：

- 查看信号
- 配置自己的股票
- 调整自己的策略参数
- 设置自己的提醒方式
- 发起自己的回测

登录安全规则：

- 连续输错 5 次密码后，账号会被临时锁定

## 5. 页面功能说明

## 5.1 监控面板

这是最常用的页面。

### 我的信号

这里会显示当天的信号列表，通常包含：

- 你自己的个人信号
- 系统的全局信号

这里显示的是“最终信号”，不是原始触发明细。

也就是说：

- 同一根 bar 最多显示 1 条结果
- `up` 和 `down` 会先互相抵消
- 如果净结果为 0，这根 bar 就不会显示
- `Lv` 取决于抵消后剩下多少个同方向指标

你可以：

- 按关键词筛选
- 按股票筛选
- 按 `1m` 或 `5m` 周期筛选
- 点击某条信号联动右侧图表

每条信号通常会显示：

- 股票名称 / 股票代码
- 触发时间
- 周期
- 最终方向
- 信号等级
- 同方向指标数量
- 合并后的规则标签
- 一句简要提示说明

### 图表

图表区可以查看当天的价格走势。

你可以：

- 切换股票
- 切换 `1m / 5m`
- 切换 `K线 / 曲线`
- 打开或关闭 `VWAP`
- 鼠标滚轮缩放
- 鼠标拖动画布平移
- 结合信号发生时间看走势变化

点击某条信号后，图表会自动切到对应股票和周期，方便你回看当时发生了什么。

### 运行状态

这里主要用来确认系统有没有正常工作。

你可以看到：

- 当前是否正在运行
- 当前监控了多少只股票
- 当前订阅了多少个周期
- 最近是否有异常
- 最近一次配置应用后是否成功

如果这里显示异常，通常说明监控没有真正跑起来，需要先排查环境或重新应用配置。

## 5.2 配置中心

这是用户日常最重要的配置页面。

### 全局股票池

仅管理员可维护。

这里添加的股票，会成为所有用户共同监控的股票。

这里要特别注意：

- 全局股票池的信号只由系统的 `global profile` 统一产出
- 所有用户看到的全局股票信号是一致的
- 普通用户自己的规则覆盖，不会影响全局股票池的信号结果

适合放：

- 团队统一关注的核心标的
- 每天都要盯的主股票

例子：

- 管理员把 `HK.00700`、`HK.00981` 加入全局股票池
- 那么所有用户登录后，都会收到这两只股票的监控结果

### 我的自选池

每个用户都可以维护自己的股票池。

这里添加的股票只对自己生效，不影响别人。

当前版本里，个人规则覆盖也只会作用在这里。

适合放：

- 你今天临时特别关注的股票
- 你个人偏好的观察标的

例子：

- 全局股票池里已经有 `HK.00700`
- 你自己再加一个 `HK.01810`
- 那你看到的监控范围就是：`HK.00700 + HK.01810`
- 其中 `HK.00700` 走系统统一规则，`HK.01810` 可以叠加你自己的规则覆盖

### 默认规则

这里展示系统当前的基础策略配置。

你可以把它理解成：

- 系统默认怎样判断“值得提醒”
- 每条规则现在的灵敏度是多少

这个区域通常是只读查看，方便你理解当前系统的默认标准。

下面是每条默认规则的简要说明。

#### 1. 开盘区间突破 `open_range_breakout`

这条规则主要看：

- 开盘后一小段时间形成的价格区间
- 后续价格有没有向上或向下突破这个区间
- 突破时量能是否明显放大

适合场景：

- 早盘快速定方向
- 消息刺激后的趋势启动

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用这条规则 |
| `range_minutes` | 开盘区间取前多少分钟。数值越小，越容易更早出现突破提醒 |
| `volume_mult` | 突破时成交量至少要达到“平均量的多少倍”。数值越大，提醒越少，但通常更严格 |
| `tf_overrides` | 针对 `1m` 或 `5m` 单独设置不同参数 |

#### 2. VWAP 偏离 `vwap_deviation`

这条规则主要看：

- 当前价格离盘中均价（VWAP）有多远
- 这种偏离是否已经明显到值得提醒
- 偏离发生时是否伴随放量

适合场景：

- 强势拉升
- 快速下杀
- 明显脱离盘中均衡价

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `atr_period` | 用多长窗口估算波动幅度 |
| `dev_atr_mult` | 偏离多少倍波动幅度才算有效。数值越小，越敏感 |
| `volume_mult` | 偏离出现时，对成交量的要求 |
| `vol_avg_period` | 平均成交量的参考窗口长度 |
| `tf_overrides` | 不同周期单独设置 |

#### 3. 波动压缩后放大 `squeeze_breakout`

这条规则主要看：

- 之前一段时间是不是很平静
- 之后波动有没有明显放大
- 放大时是否伴随放量

适合场景：

- 长时间横盘后突然启动
- 盘中从沉闷转向活跃

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `atr_period` | 用于衡量波动的基础窗口 |
| `lookback` | 往前看多少根 K 线来判断“最近是否处于低波动” |
| `low_quantile` | 用多严格的标准判断“低波动”。数值越小，代表越强调“先压缩得很明显” |
| `volume_mult` | 放大时要求的量能倍数 |
| `tf_overrides` | 周期单独设置 |

#### 4. RSI 超买超卖 `rsi_extreme`

这条规则主要看：

- 当前走势是不是已经过热
- 或者是不是已经过度下跌

适合场景：

- 提醒短线过热
- 提醒短线超跌
- 辅助判断是否接近情绪极端

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `period` | RSI 的计算窗口 |
| `overbought` | 超买阈值。越低越容易触发“过热”提醒 |
| `oversold` | 超卖阈值。越高越容易触发“超跌”提醒 |

#### 5. 突破回踩确认 `break_retest`

这条规则主要看：

- 价格突破关键位置后
- 有没有回踩但不破
- 然后再次走强或走弱

适合场景：

- 过滤假突破
- 寻找二次确认机会

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `retest_max_bars` | 突破后，最多允许等多少根 K 线来完成回踩确认 |
| `retest_tolerance_pct` | 回踩时允许的误差范围。数值越大，系统越宽松 |

#### 6. 量价背离 `volume_price_divergence`

这条规则主要看：

- 价格还在创新高或创新低
- 但成交量没有跟上，甚至开始走弱

适合场景：

- 提醒冲高乏力
- 提醒下跌衰竭
- 辅助判断趋势是否可能转弱

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `lookback` | 往前比较多少根 K 线 |
| `volume_mult` | 当前量能低于平均量的判定倍数。数值越小，越强调“明显缩量” |

#### 7. 昨日高低点突破 `prev_day_break`

这条规则主要看：

- 今天价格有没有突破昨天的高点
- 或跌破昨天的低点
- 突破时有没有量能配合

适合场景：

- 关键位置突破
- 情绪延续
- 前一日重要高低点被重新定价

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `volume_mult` | 突破昨日高低点时要求的量能倍数 |

#### 关于 `tf_overrides`

如果某条规则里有 `tf_overrides`，意思是：

- 这条规则可以给 `1m` 和 `5m` 设不同参数
- 你可以让短周期更敏感，长周期更稳健

例子：

- `1m` 用较低阈值，争取更早发现机会
- `5m` 用较高阈值，减少杂讯

### 我的规则覆盖

这是最重要也最容易觉得复杂的功能。

它的作用不是让你重写整套策略，而是让你只改自己关心的那一小部分。

适合的使用场景：

- 你觉得某条规则太敏感，提醒太多
- 你觉得某条规则太迟钝，错过很多机会
- 你只想调整 `1m` 周期，不动 `5m`
- 你只想关闭某一条自己不想看的规则

你可以这样理解：

- 系统先有一套默认规则
- 你在“我的规则覆盖”里写的内容，只会覆盖你改动的部分
- 没写的部分，仍然沿用默认规则
- 这些覆盖只作用在你的自选股票池，不会改动全局股票池的统一信号

例子 1：关闭 RSI 提醒

如果你不想看 RSI 超买超卖信号，可以只写：

```json
{
  "rsi_extreme": {
    "enabled": false
  }
}
```

效果：

- 你自己的账号不再收到 RSI 类提醒
- 其他用户不受影响
- 全局股票池也不受影响

例子 2：让 1 分钟信号更敏感

如果你想让 `1m` 的 VWAP 偏离更容易触发，可以把阈值调低：

```json
{
  "vwap_deviation": {
    "tf_overrides": {
      "1m": {
        "dev_atr_mult": 1.0
      }
    }
  }
}
```

效果：

- 只调整你自己的 `1m` 判定灵敏度
- `5m` 规则不变
- 全局默认规则不变

例子 3：减少开盘突破误报

如果你觉得开盘区间突破提醒太多，可以提高量能门槛：

```json
{
  "open_range_breakout": {
    "volume_mult": 2.0
  }
}
```

效果：

- 只有量能更明显放大的突破，才更容易提醒你

使用建议：

- 每次只改一两项
- 改完后观察一段时间再继续调
- 如果想恢复默认，只要把对应覆盖内容删掉或改回默认值

### 通知设置

你可以在这里设置自己想怎么接收提醒。

支持三种方式：

- `local`：本地提醒
- `telegram`：Telegram 消息
- `both`：两者都发

还可以设置：

- Telegram token
- Telegram chat id
- 是否响铃

适合的用法：

- 在办公室盯盘时，用 `local`
- 不在电脑前时，用 `telegram`
- 不想错过提醒时，用 `both`

## 5.3 用户管理

仅管理员可见。

支持功能：

- 创建新用户
- 指定用户角色
- 重置用户密码

适合场景：

- 团队里给不同成员分配独立账号
- 避免大家共用同一个提醒设置
- 每个人维护自己的股票池和策略偏好

## 5.4 策略实验室

这个页面适合做“试验”和“复盘”。

### 策略模板

策略模板可以理解为“保存好的策略版本”。

适合场景：

- 你调出了一套自己满意的参数，想保存下来
- 你想把几套不同风格的参数分别留档

例子：

- 模板 A：偏激进，适合高波动日
- 模板 B：偏保守，适合减少噪音

这样后续你回看时，就知道某次回测是基于哪套思路做的。

### 本地回测

回测用于验证某个日期、某只股票上，这套规则当时表现如何。

你只需要填写：

- 股票代码
- 日期

系统会帮你回放那一天的数据，并给出结果摘要。

回测结果通常可以帮助你回答这些问题：

- 这套参数是不是提醒太多
- 这些提醒之后，价格有没有真的走出来
- `1m` 和 `5m` 哪个更有效

例子：

- 你最近把 VWAP 偏离调得更敏感
- 想知道这样会不会带来太多无效信号
- 你就可以选 `HK.00981` 和某个交易日发起回测
- 然后对比回测摘要，看命中率和后续波动情况

### 分析概览

这里会汇总一些总体统计信息，帮助你快速了解使用情况，例如：

- 一共出现了多少条信号
- 哪些规则触发最多
- 回测任务目前有多少在排队、运行中、已完成、失败

这个区域更适合用来看整体情况，不是逐条交易复盘工具。

## 6. 常见使用流程

## 6.1 新用户第一次使用

1. 登录系统
2. 先看监控面板，熟悉信号列表和图表
3. 到“我的自选池”里添加自己关注的股票
4. 到“通知设置”里选择提醒方式
5. 如果觉得提醒太多或太少，再去调整“我的规则覆盖”

## 6.2 想减少无效提醒

可以优先尝试：

- 提高某条规则的触发门槛
- 关闭自己不需要的规则
- 只调整 `1m`，先不要动 `5m`

一个实用思路是：

- 先减少最吵的那类提醒
- 观察两三个交易日
- 再决定是否继续调整

## 6.3 想提高捕捉速度

可以优先尝试：

- 降低某条规则的阈值
- 让 `1m` 规则更敏感
- 增加自己特别关注的股票到“我的自选池”

但要注意：

- 更敏感通常意味着更多提醒
- 更多提醒不一定代表更高质量

## 6.4 想验证调整是否有效

推荐做法：

1. 先修改“我的规则覆盖”
2. 记录自己改了什么
3. 用“本地回测”验证几个典型交易日
4. 再决定是否长期采用这组参数

## 7. 使用建议

- 如果你是第一次用，先不要一次改很多参数
- 优先从股票池和提醒方式开始
- 规则参数建议一次只改一个方向
- 对结果不确定时，用回测先验证
- 团队使用时，建议管理员维护公共股票池，个人维护自己的偏好配置

## 8. 注意事项

- 本项目用于提示，不用于自动交易
- 提醒结果不能替代人工判断
- 参数调得越敏感，提醒通常会越多
- 不同股票、不同波动环境下，适合的参数可能不一样
- 如果修改了股票池或策略后没有看到预期效果，建议检查是否已经重新应用配置
