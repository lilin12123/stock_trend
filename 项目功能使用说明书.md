# 项目功能使用说明书

当前版本：`V1.0.0`

## 1. 适用对象

本说明书面向日常使用本系统的用户。

阅读本说明书时，重点关注以下内容：

- 怎么注册和登录
- 怎么看盘中信号
- 怎么添加自己的观察股票
- 怎么调整自己的规则偏好
- 怎么做回测和看结果

本说明书仅介绍日常使用过程中需要了解的功能和操作方法。

## 2. 产品说明

这是一个以**日内股票趋势观察**为核心的网页控制台，主要用于盘中盯盘，并为日内交易提供辅助指引。

当前版本支持的市场范围：

- 港股
- 美股

当前状态说明：

- A 股支持正在研发中

> **提示：当前版本仍处于测试中，可用指标数量较少，指标逻辑也相对简单，请将提示结果作为辅助参考，不建议单独作为交易决策依据。**

体验地址：

- [https://trend.freechatee.com/](https://trend.freechatee.com/)

它会持续监控股票，在满足预设条件时给出提示，帮助你更快发现：

- 突破
- 趋势加强
- 短线过热或过冷
- 波动突然放大
- 关键价位突破

你可以把它理解成：

- 一个持续观察日内趋势变化的工具
- 一个帮助你快速发现盘中异动与趋势机会的辅助系统
- 一个为日内交易提供参考线索的信号面板

它的定位是 **提示工具**，不是自动交易系统，不会替你下单。

## 3. 主要功能

- 注册普通账号并登录
- 查看系统生成的盘中信号
- 查看图表，并把信号和价格走势对照起来看
- 查看全局股票池
- 维护自己的 1 支自选股票
- 保存自己的规则覆盖
- 设置自己的通知方式
- 发起本地回测
- 查看最近一次回测的详细结果
- 查看系统统计概览

## 4. 首次使用

### 4.1 注册

如果系统开启了自助注册，你可以在登录页右侧直接注册。

注册时只需要填写：

- 用户名
- 密码

注册页会提示：

- 当前是否开放注册
- 还剩多少普通用户名额

如果系统已经达到注册上限，或者暂时关闭了自助注册，你需要联系系统维护方开通账号。

### 4.2 登录

登录后会进入系统主界面。

登录有一个安全限制：

- 连续输错 5 次密码后，账号会被临时锁定

## 5. 页面说明

### 5.1 监控面板

这是你最常用的页面。

### 我的信号

这里会显示系统整理后的“最终信号”，不是原始触发明细。

你看到的每条信号，已经经过了合并和方向判断：

- 同一根 K 线最终只显示 1 条结果
- 如果同一时刻既有看多指标也有看空指标，它们会先互相抵消
- 只有抵消后还有净方向，才会显示出来
- `Lv` 等级代表抵消后还剩下多少个同方向指标

你可以在这里做的事：

- 按关键词筛选
- 按股票筛选
- 按 `1m / 5m` 周期筛选
- 点击某条信号，让图表自动切到对应股票和周期

每条信号通常会显示：

- 股票名称 / 股票代码
- 触发时间
- 周期
- 最终方向
- 信号等级
- 同方向指标数量
- 合并后的规则标签
- 简短提示语

### 图表

图表区域用来回看盘中走势。

你可以：

- 切换股票
- 切换 `1m / 5m`
- 切换 `K 线 / 曲线`
- 打开或关闭 `VWAP`
- 用滚轮缩放
- 用鼠标拖动画布平移
- 对照信号发生时间，看价格后续怎么走

点击左侧某条信号后，图表会自动跳到对应股票和周期。

### 运行状态

这个区域用于确认系统有没有正常运行。

你能看到：

- 当前是否在运行
- 当前监控了多少只股票
- 当前订阅了多少个周期
- 最近是否有异常
- 最近一次配置应用是否成功

如果这里出现报错，通常说明数据接入或运行状态有问题，这时先不要急着看信号，应该先确认系统是否正常。

### 5.2 配置中心

这是普通用户第二常用的页面。

#### 全局股票池

这里展示系统统一监控的股票。

这一部分主要用于查看系统统一监控的股票范围。

这些股票通常由系统统一设置，所有用户都能看到它们对应的信号。

你可以把它理解成：

- 团队公共观察池
- 系统默认会盯的股票

#### 我的自选池

这里是你自己的观察股票。

当前版本有一个重要限制：

- 每位用户当前最多只能维护 `1` 支自选股票

也就是说：

- 如果你还没有添加，可以新增 1 支
- 如果你已经有了，再添加另一支会失败

代码格式建议按系统惯例填写，例如：

- `HK.00700`
- `US.NVDA`

适合的使用方式：

- 把你当天最想重点观察的那一支股票放进来
- 不要把它当成大股票池来用

关于“用户自选信号”，还需要特别注意下面几点：

- 系统当前只会为你的这 `1` 支自选股票生成个人信号
- 全局股票的信号仍然按系统统一规则生成
- 你的个人规则覆盖，主要作用在这 `1` 支自选股票上
- 如果你的自选股票和全局股票重复，页面上同一时刻的结果可能会被合并展示为 `1` 条最终信号

#### 默认规则

这里展示的是系统当前的基础规则。

它的作用是让你知道：

- 系统默认是怎样判断“值得提醒”的
- 当前每条规则的大致灵敏度如何

这部分通常作为参考阅读，不需要你在这里直接编辑。

#### 我的规则覆盖

这是日常使用中最重要、也最容易误解的功能。

它的作用不是让你重写整套策略，而是让你在默认规则的基础上，只修改自己关心的那一小部分。

你可以把它理解成：

- 默认规则是“系统统一标准”
- 我的规则覆盖是“只对我自己额外生效的个性化调整”

当前版本有一个非常重要的边界：

- 你的规则覆盖 **只作用于你自己的自选股票**
- 不会改变全局股票池的信号结果

也就是说：

- 全局股票池的信号，所有用户看到的是同一套结果
- 你自己的规则覆盖，主要用于你那 1 支个人自选股票

适合修改的场景：

- 你觉得某类信号太多，想关掉
- 你觉得某类信号太迟，想调得更敏感
- 你只想微调 `1m`，不动 `5m`

例子 1：关闭 RSI 提醒

```json
{
  "rsi_extreme": {
    "enabled": false
  }
}
```

效果：

- 你个人自选股票上，不再提示 RSI 超买/超卖
- 全局股票池不受影响

例子 2：让 1 分钟 VWAP 偏离更敏感

```json
{
  "vwap_deviation": {
    "tf_overrides": {
      "1m": {
        "dev_atr_mult": 1.6
      }
    }
  }
}
```

效果：

- 你的个人自选股票在 `1m` 周期上更容易出现 VWAP 偏离提醒
- `5m` 不变
- 全局股票池不变

使用建议：

- 每次只改一两项
- 改完后观察几天
- 配合回测验证，不要只凭感觉调整

#### 通知配置

当前可用的通知方式比较简单。

你可以选择：

- `关闭所有通知`
- `仅本地通知`

你还可以设置：

- 是否响铃

当前版本里，用户账号默认只使用本地通知能力。

#### 应用配置

当你改完下面这些内容后，建议点击左侧栏的 `应用配置`：

- 我的自选池
- 我的规则覆盖
- 通知设置

这样系统会重新应用运行中的配置，你的新设置能更快生效。

### 5.3 策略实验室

这一页最值得重点使用的是回测功能。

#### 本地回测

你可以在这里验证：

- 某只股票
- 某一天
- 在某组规则和条件下
- 实际表现如何

你需要填写或选择：

- 股票
- 日期
- 回测周期：`1m`、`5m`，可以多选
- 测试通过条件

这里的股票下拉通常来自：

- 全局股票池
- 你的那 1 支个人自选股票

回测功能当前有几个重要限制：

- 只能回测当前下拉框中能选到的股票，不能随意输入任意股票
- 只能选择 `1m` 和 `5m` 周期
- 至少要选择 `1` 个周期才能提交
- 回测使用的是“默认规则 + 你的规则覆盖”合并后的结果
- 这一点与实时自选信号不同：回测时，你的规则覆盖也会作用到你选中的全局股票
- 回测只用于验证信号触发后的波动表现，不代表交易盈亏结果

测试通过条件包括：

- 观察窗口（分钟）
- 最小波动（%）

它的意思是：

- 系统先找到当天触发过的信号
- 再看信号出现后的一段时间里，价格是否真的走出了足够大的波动

例子：

- 你把 `1m` 的观察窗口设成 `15` 分钟
- 最小波动设成 `0.5%`

那系统的判断方式就是：

- 某条 `1m` 信号出现后
- 往后看 15 分钟
- 只要最大上涨或最大回撤有一边达到 `0.5%`
- 这条信号就算“通过”

#### 回测结果说明

提交回测后，页面会展示两部分内容：

- 回测 Summary
- 触发信号表格

你能看到的信息包括：

- 回测状态：排队中、运行中、已完成、失败
- 总共评估了多少条信号
- 有多少条达到通过条件
- 通过率
- `1m` 和 `5m` 分开统计后的结果
- 每条信号对应的最大上涨、最大回撤、观察窗口和最终结果

一个很重要的当前行为是：

- 系统当前只保留你最近一次回测结果
- 你再次提交回测时，旧的回测记录会被新的替换
- 回测结果不会自动帮你保存多版本对比记录

所以更合适的使用方式是：

- 一次只验证一个问题
- 看完结果后再改参数做下一轮

#### 分析概览

这个区域展示的是简单统计，不是深度研究报告。

它通常会告诉你：

- 你当前可见范围内累计有多少条信号
- 哪些规则触发最多
- 你的回测任务状态分布

适合用来快速了解整体情况，不适合替代逐条复盘。

#### 策略模板

页面里你可能会看到“策略模板”区域。

更建议把它当成：

- 参考区
- 展示区

日常重点仍然是：

- 调整自己的规则覆盖
- 进行自己的回测

如果保存模板失败，说明当前账号不适合直接使用这一能力，此时建议以规则覆盖和回测为主。

## 6. 使用时需要特别注意的事项

这几条非常关键，建议直接记住。

### 1. 自选池最多只能保留 1 支股票

这是当前版本的产品限制，不是操作问题。

### 2. 规则覆盖只影响自己的自选股票

它不会改变全局股票池的信号结果。

补充说明：

- 如果你的自选股票与全局股票重复，页面中的最终信号可能会做合并展示

### 3. 当前通知方式以本地通知为主

如页面未开放其他通知方式，请以当前可选项为准。

### 4. 信号列表展示的是“最终合并结果”

不是每个指标原始逐条列出来的列表。

### 5. 新回测会覆盖上一次回测结果

如果你想对比两次参数，请自己先记录关键结果。

### 6. 回测的规则生效范围与实时信号不同

- 实时个人信号主要围绕你的自选股票生效
- 回测时，你的规则覆盖会直接参与所选股票的回测计算
- 因此不要把“实时展示效果”和“回测结果范围”简单理解成完全一致

## 7. 推荐使用方式

建议按下面两种方式来使用。

### 方式一：先关注上一个交易日的信号，再进入实时使用

这种方式适合：

- 第一次接触系统
- 还不熟悉各类信号含义
- 希望先建立“这套信号大概会在什么情况下出现”的直觉

建议步骤：

1. 先登录系统，不急着修改规则
2. 打开监控面板，查看最近留下来的信号结果
3. 重点观察上一个交易日里哪些股票出现了较高等级信号
4. 点击信号，联动查看对应图表，理解信号出现前后价格是怎么走的
5. 先记住几类你最常看到的信号，例如开盘区间突破、VWAP 偏离、RSI 超买超卖
6. 再到配置中心添加自己的 1 支重点股票
7. 开启本地通知，开始进入实时观察

这种方式的好处是：

- 先建立对信号的基本理解
- 不容易一上来就把参数调乱
- 更适合先学会“怎么看”，再决定“怎么调”

### 方式二：直接用回测功能做调试

这种方式适合：

- 已经有明确想法
- 已经知道自己想减少哪类噪音，或增强哪类信号
- 想快速验证某组参数是否更合适

建议步骤：

1. 先在配置中心设置自己的 1 支重点股票
2. 在“我的规则覆盖”中只修改 1 到 2 个参数
3. 进入策略实验室，选择股票、日期和回测周期
4. 设置测试通过条件，例如观察窗口和最小波动
5. 提交回测，查看 Summary 和信号表格
6. 对照通过率、最大上涨、最大回撤，判断这次调整是否更符合你的预期
7. 如果结果不理想，就继续小幅调整，再做下一轮回测

这种方式的好处是：

- 可以更快验证参数是否有效
- 不需要完全依赖主观感觉
- 更适合做小步调试和迭代

### 使用建议

如果你是第一次使用，建议优先采用“方式一”。  
如果你已经比较熟悉这些规则，或者已经有明确的调参目标，建议直接采用“方式二”。

## 8. 默认规则说明

下面是系统当前默认规则的简要说明，方便你理解“系统现在在看什么”。

> **提示：当前默认规则仍处于测试阶段，规则数量较少，判断逻辑以简洁可验证为主，覆盖面和精细度都还有限。**

当前这套默认配置的整体取向是：

- `1m` 周期比 `5m` 更保守，优先减少高频噪音
- 主信号更偏趋势确认，不强调抄顶摸底
- `squeeze_breakout` 默认只保留在 `5m`
- `rsi_extreme` 默认只保留在 `5m`，并使用更严格阈值
- `volume_price_divergence` 默认关闭，避免干扰主信号流

### 8.1 开盘区间突破 `open_range_breakout`

看的是：

- 开盘后一段时间形成的区间
- 后续是否向上或向下突破
- 突破时是否伴随放量

适合场景：

- 早盘定方向
- 消息刺激后的趋势启动

当前默认设置里：

- `1m` 会比 `5m` 要求更长的开盘区间和更高的量能门槛

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `range_minutes` | 开盘区间取前多少分钟 |
| `volume_mult` | 突破时成交量至少要达到平均量的多少倍 |
| `tf_overrides` | 对 `1m / 5m` 单独设置不同参数 |

### 8.2 VWAP 偏离 `vwap_deviation`

看的是：

- 当前价格离盘中均价有多远
- 这种偏离是否已经大到值得提醒
- 偏离时有没有放量

适合场景：

- 强势拉升
- 快速下杀
- 明显脱离盘中均衡价

当前默认设置里：

- `1m` 的偏离倍数和量能要求高于 `5m`
- 这样做是为了减少瞬时拉扯带来的误报

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `atr_period` | 用多长窗口估算波动幅度 |
| `dev_atr_mult` | 偏离多少倍波动才算有效 |
| `volume_mult` | 偏离出现时，对成交量的要求 |
| `vol_avg_period` | 平均成交量的参考窗口 |
| `tf_overrides` | 对 `1m / 5m` 单独设置不同参数 |

### 8.3 波动压缩后放大 `squeeze_breakout`

看的是：

- 前面是不是处于低波动状态
- 后面是否突然开始放大
- 放大时有没有量能配合

适合场景：

- 横盘后启动
- 沉闷行情突然变活跃

当前默认设置里：

- 该规则默认只在 `5m` 启用
- `1m` 默认关闭，避免把普通波动放大误判成有效启动

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `atr_period` | 衡量波动的基础窗口 |
| `lookback` | 往前看多少根 K 线 |
| `low_quantile` | 低波动判断标准的严格程度 |
| `volume_mult` | 放大时要求的量能倍数 |
| `tf_overrides` | 对 `1m / 5m` 单独设置不同参数 |

### 8.4 RSI 超买超卖 `rsi_extreme`

看的是：

- 当前走势是不是过热
- 或是不是过度下跌

适合场景：

- 提醒短线情绪极端
- 辅助判断回调或反弹风险

当前默认设置里：

- 该规则默认只在 `5m` 启用
- 默认阈值比旧版本更严格，为 `75 / 25`
- 它更偏“风险提醒”，不是默认主趋势信号

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `period` | RSI 的计算窗口 |
| `overbought` | 超买阈值 |
| `oversold` | 超卖阈值 |

### 8.5 突破回踩确认 `break_retest`

看的是：

- 价格突破之后
- 有没有回踩但不破
- 然后再次沿原方向走

适合场景：

- 过滤假突破
- 找二次确认机会

当前默认设置里：

- 回踩确认的等待窗口比旧版本更短
- 目的是减少拖很久之后才出现的“迟到确认”

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `retest_max_bars` | 最多允许多少根 K 线完成回踩确认 |
| `retest_tolerance_pct` | 回踩允许的误差范围 |

### 8.6 量价背离 `volume_price_divergence`

看的是：

- 价格还在创新高或创新低
- 但量能没有跟上

适合场景：

- 提醒冲高乏力
- 提醒趋势可能衰减

当前默认设置里：

- 该规则默认关闭
- 如果你特别关心背离提醒，可以在自己的规则覆盖里单独开启

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `lookback` | 往前比较多少根 K 线 |
| `volume_mult` | 当前量能低于平均量的判定倍数 |

### 8.7 昨日高低点突破 `prev_day_break`

看的是：

- 今天是否突破昨天高点
- 或跌破昨天低点
- 突破时是否有量能配合

适合场景：

- 关键位置突破
- 情绪延续

当前默认设置里：

- `1m` 的量能要求高于 `5m`
- 这样做是为了减少昨日关键位附近的来回穿越噪音

常见参数：

| 参数 | 说明 |
|---|---|
| `enabled` | 是否启用 |
| `volume_mult` | 突破昨日高低点时要求的量能倍数 |

## 9. 使用建议

- 先把系统当成“提醒器”，不要一上来就大改参数
- 如果你是第一次使用，建议先保留默认策略，先观察 `1m` 和 `5m` 的节奏差异
- 如果你只想做一件事，优先把自己的那 1 支重点股票配置好
- 如果信号太多，先关掉一类你最不关心的规则
- 如果你不确定某个改动好不好，用回测说话
- 新回测会覆盖旧回测，所以重要结果要自己留痕
